## Quick Start

### Option 1: CSS Variables (Recommended)

```css
/* In your global CSS */
@import './design-system/tokens.css';

/* Usage */
.my-component {
  background: var(--bg-surface);
  color: var(--text-primary);
  border-radius: var(--radius-card);
  padding: var(--card-padding);
}
```

### Option 2: Tailwind Config

```js
// tailwind.config.js
const tokens = require('./design-system/tokens.tailwind.js');

module.exports = {
  theme: {
    extend: {
      ...tokens,

  },
};
```


## Token Architecture

```
┌─────────────────────────────────────────────────────────────┐
│  LAYER 3: COMPONENT                                         │
│  glass-panel, card, badge, progress-bar                     │
├─────────────────────────────────────────────────────────────┤
│  LAYER 2: SEMANTIC                                          │
│  bg-base, text-primary, accent-primary, success, etc.       │
├─────────────────────────────────────────────────────────────┤
│  LAYER 1: CORE                                              │
│  neutral-950, cyan-400, space-4, radius-lg, etc.            │
└─────────────────────────────────────────────────────────────┘
```

**Rule:** Components use SEMANTIC tokens. Never use CORE tokens directly.

## Migration Path

To migrate existing components:

1. Import `tokens.css` in your main CSS file
2. Replace hardcoded hex values with CSS variables
3. Replace inline styles with component classes
4. Use semantic tokens for all colors

Example migration:

```tsx
// BEFORE
<div className="bg-[#0f172a] border border-white/5 rounded-2xl p-8">

// AFTER
<div className="card">
// Or with CSS variables:
<div style={{ background: 'var(--bg-surface)', borderRadius: 'var(--radius-card)' }}>
```
